# 🐰 らびの自動投稿：メンテナンスガイド

自動投稿エンジン (`scripts/auto_latest_news.cjs`) を新しく整備し、メンテナンスしやすくしました！

## 🛠️ 便利なコマンド

`package.json` に以下のスクリプトを追加しました：

- **ドライラン（テスト実行）**:
  ```bash
  npm run labi:dry-run
  ```
  実際に投稿はせず、何を投稿しようとしているかを確認できます。

- **手動投稿**:
  ```bash
  npm run labi:post
  ```
  今すぐ新しいニュースを投稿したい時に使います。

## 📝 ログの確認

実行結果はプロジェクトのルートにある `labi_auto_post.log` に自動的に記録されます。
「最近ちゃんと投稿されてるかな？」と思ったら、このファイルを確認してね。

## 🔍 主要な修正点 (2026/03/21)

1. **テキストのクリーンアップ強化**:
   - `<![CDATA[` や `]]>` などのRSS特有のゴミを自動で取り除くようにしました。
   - `&amp;` や `&#45;` などの文字化け（HTMLエンティティ）も自動でデコードします。

2. **堅牢なエラーハンドリング**:
   - RSSの取得に失敗しても、他のニュースの投稿を止めずに続行します。
   - 重複チェックを強化し、同じニュースが何度も投稿されるのを防ぎます。

3. **カテゴリの多様性**:
   - ニュースばかりにならないよう、レビューやコラムなどもバランスよく選ぶロジックになっています。

これからも、らびが元気にニュースを届けてくれるはずらびっ！🥕🐰
